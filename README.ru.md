# FastAPI ChatAPI

Чат-приложение в реальном времени, построенное на FastAPI, WebSockets, PostgreSQL, Redis и Celery.

Этот проект демонстрирует, как объединить REST API, JWT-аутентификацию, WebSocket-коммуникацию, фоновые задачи, обмен файлами и обработку изображений в современном backend для чата.

 

## Возможности

### Аутентификация
- Регистрация пользователей
- Хеширование паролей с помощью bcrypt
- JWT-аутентификация
- Защищённые маршруты

### Профили пользователей
- Просмотр и редактирование профиля
- Загрузка аватаров
- Автоматическое изменение размера аватаров через Celery + Pillow
- Кэширование ответов профиля

### Комнаты
- Создание публичных и приватных комнат
- Вступление и выход из комнат
- Просмотр участников комнаты
- Просмотр списка публичных комнат

### Сообщения
- Обмен сообщениями в реальном времени через WebSocket
- Получение истории сообщений
- Курсорная пагинация
- Индикаторы набора текста
- Уведомления о входе и выходе пользователей
- Отслеживание пользователей онлайн

### Обмен файлами
- Загрузка файлов в комнаты
- Генерация миниатюр изображений
- Уведомления о новых файлах в реальном времени

### Фоновые задачи
- Celery с Redis в качестве брокера
- Обработка аватаров
- Генерация миниатюр изображений
- Отложенная отправка сообщений (в разработке)

# Локальный запуск

## Требования

- Docker
- Docker Compose
- Python 3.12+
- PostgreSQL (если запускать без Docker)

 

## Переменные окружения

Создайте файл `.env`:

```env
DATABASE_URL=postgresql://chatapp_user:your_password_here@db:5432/chatapp
JWT_SECRET=your-secret-key
REDIS_URL=redis://redis:6379/0
UPLOAD_PATH=uploads
```

 

## Запуск через Docker

Сборка и запуск:

```bash
docker compose up --build
```

Последующие запуски:

```bash
docker compose up -d
```

Выполнение миграций:

```bash
docker compose run --rm app uv run alembic upgrade head
```

Просмотр логов Celery Worker:

```bash
docker compose logs -f worker
```

Перезапуск Celery Worker:

```bash
docker compose restart worker
```

Остановка всех сервисов:

```bash
docker compose down -v
```

 

## Настройка базы данных без Docker

### Создание базы данных

Подключитесь под пользователем PostgreSQL:

```bash
psql -U postgres
```

Создайте базу данных и пользователя:

```sql
CREATE DATABASE chatapp;

CREATE USER chatapp_user WITH PASSWORD 'your_password_here';

GRANT ALL PRIVILEGES ON DATABASE chatapp TO chatapp_user;

\q
```

Выдайте права на схему:

```bash
psql -U postgres -d chatapp -h localhost
```

```sql
GRANT ALL ON SCHEMA public TO chatapp_user;

\q
```

### Выполнение миграций

Инициализация Alembic:

```bash
alembic init alembic
```

Создание миграции:

```bash
alembic revision --autogenerate -m "initial"
```

Применение миграции:

```bash
alembic upgrade head
```

# Архитектурные решения

## Почему FastAPI + Async?

Большинство операций в чат-приложении ограничены скоростью ввода-вывода (I/O), а не вычислениями процессора.

Пока пользователи подключены, сервер в основном ожидает:

- Новые сообщения
- Ответы от базы данных
- Сетевые события

Асинхронные эндпоинты и WebSocket позволяют обслуживать тысячи одновременных подключений без блокировки потоков.

 

## JWT-аутентификация

Схема работы:

```text
Регистрация
      ↓
Хеширование пароля
      ↓
Сохранение в БД
      ↓
Вход
      ↓
JWT-токен
      ↓
Авторизованные запросы
```

Преимущества:

- Stateless-аутентификация
- Масштабируемость
- Хорошо подходит для REST API
- Удобна для авторизации WebSocket-подключений

 

## Курсорная пагинация

Пагинация через OFFSET:

```sql
OFFSET 50 LIMIT 50
```

плохо подходит для чатов, потому что новые сообщения постоянно сдвигают границы страниц.

Вместо этого используется:

```http
GET /messages?before=123
```

Пагинация привязывается к конкретному сообщению, что предотвращает пропуски и дубликаты.

 

## WebSockets против REST

| Возможность | REST | WebSocket |
|------------|------|-----------|
| Постоянное соединение | ❌ | ✅ |
| Push от сервера | ❌ | ✅ |
| Обновления в реальном времени | ❌ | ✅ |
| Накладные расходы | Высокие | Низкие |
| Подходит для чата | ❌ | ✅ |

REST используется для:

- Аутентификации
- Управления комнатами
- Получения истории сообщений
- Загрузки файлов

WebSocket используется для:

- Сообщений в реальном времени
- Уведомлений о присутствии
- Индикаторов набора текста
- Уведомлений о новых файлах

 

# Фоновые задачи Celery

### Обработка аватаров

После загрузки аватара:

```python
process_avatar.delay(file_path)
```

Worker:

1. Получает задачу из Redis
2. Загружает изображение
3. Изменяет размер до 256×256
4. Сохраняет оптимизированную версию

### Генерация миниатюр

При загрузке изображения:

```python
generate_thumbnail.delay(file_path)
```

Миниатюры создаются асинхронно, поэтому загрузка файлов не блокирует пользователя.

# Примечания по разработке

Celery загружает код задач в память при запуске.

Если код задачи изменился:

```bash
docker compose restart worker
```

необходимо перезапустить worker, чтобы он загрузил новую версию кода.

FastAPI в режиме разработки запускается с:

```bash
--reload
```

поэтому изменения в API подхватываются автоматически.
